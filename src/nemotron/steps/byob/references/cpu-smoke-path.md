# CPU-only / mock-endpoint smoke path

BYOB and translation are GPU-class workloads — Curator's semantic
deduplication pulls in `cudf-cu12` / `cuml-cu12` (Linux/x86_64 only) and the
generation flow expects a hosted LLM endpoint. This page documents the
fastest way to validate **config wiring** on a developer laptop **before
requesting GPU hardware**.

The path is layered: each step is cheap and runs locally on macOS, Windows,
or Linux without GPUs. Stop at the first signal that catches the issue.

## 1. Static skill validation

Run the in-tree validator. It checks required files, frontmatter shape,
config parsing, and pattern indices. No model calls, no RAPIDS, no GPU.

```bash
python -m nemotron.steps.byob.scripts.validate
```

## 2. Catalog smoke

Confirm the BYOB step is registered and the CLI can introspect it. Both
commands work on a base `uv sync` (no `byob` extra needed).

```bash
uv run nemotron steps list --json | jq '.[] | select(.id == "byob")'
uv run nemotron steps show byob
uv run nemotron byob --list-families
```

If `--list-families` prints `mcq`, the registry import path is sound.

## 3. Notebook dry-preview

The BYOB notebook defaults to command-preview mode (`RUN_BYOB = False`):

```bash
uv run jupyter lab use-case-examples/build-your-own-benchmark/build_mcq_benchmark.ipynb
```

Run cells 1–4. The example writes a working config from the packaged
`config/default.yaml` and prints the exact CLI command that would run. This
catches config-shape regressions without pulling Curator or hitting an
endpoint.

## 4. CPU-only end-to-end with a mock LLM

Use [`config/cpu_smoke.yaml`](../config/cpu_smoke.yaml). It disables every
stage that requires RAPIDS or a large embedding model — semantic
deduplication, semantic outlier detection, coverage check, distractor
expansion — so the run is wire-only.

`config/cpu_smoke.yaml` reads the LLM base URL from `BYOB_MOCK_LLM_URL` and
the API key from `BYOB_MOCK_API_KEY`. Point both at any local OpenAI-
compatible mock — for example `vllm --served-model-name=...` against a tiny
CPU model, [`promptfoo`'s mock provider][promptfoo-mock], or
[`litellm-proxy`][litellm-proxy] in mock mode. The point is shape-validation
of the request/response loop, not output quality.

```bash
# example: minimal mock server returning canned chat completions
export BYOB_MOCK_LLM_URL=http://localhost:8000/v1
export BYOB_MOCK_API_KEY=mock-key

uv run nemotron byob --family mcq --stage prepare \
    --config src/nemotron/steps/byob/config/cpu_smoke.yaml

uv run nemotron byob --family mcq --stage generate \
    --config src/nemotron/steps/byob/config/cpu_smoke.yaml
```

Successful completion produces a `seed.parquet` and a `benchmark_raw.parquet`
under `/tmp/byob/cpu_smoke/byob_mcq_cpu_smoke/`. That confirms config wiring
and dispatcher behavior end-to-end without a GPU.

[promptfoo-mock]: https://www.promptfoo.dev/docs/providers/openai/
[litellm-proxy]: https://docs.litellm.ai/docs/proxy/quick_start

## 5. Real endpoint, CPU executor

Once the local mock works, point the same config at a real hosted endpoint
and run on a CPU-only Lepton/Slurm shape using the `lepton_byob_cpu`
profile generated by `env/env_toml`. Keep `semantic_deduplication_config.enabled=false`
and friends until the run promotes to GPU hardware with the `byob` extra
installed.

```bash
export NEMOTRON_ENV_FILE=env.lepton.toml
export BYOB_MOCK_LLM_URL=https://integrate.api.nvidia.com/v1
export BYOB_MOCK_API_KEY=$NVIDIA_API_KEY

uv run nemotron steps run byob \
    -c cpu_smoke \
    --batch lepton_byob_cpu \
    stage=all
```

Promote to `lepton_byob_full` only after CPU smoke passes; that profile
provisions GPU + RAPIDS for semantic dedup and outlier detection.

## When the CPU-only path is not enough

Switch to `config/tiny.yaml` (small but full pipeline) on a Linux/x86_64
host with the `byob` extra installed:

```bash
uv sync --extra byob
uv run nemotron byob --family mcq --stage all \
    --config src/nemotron/steps/byob/config/tiny.yaml
```

`tiny.yaml` keeps semantic deduplication and outlier detection enabled, so
it requires `cudf-cu12` / `cuml-cu12` and a sentence-transformers download.
